Мы делаем кроссплатформенный открыватель ссылок.
Это будет программа, которая будет определяться как дефолтный браузер.
Приложение должно работать на MacOS, Windows и Linux.
Написанное на языке Kotlin с использованием фреймворков Compose Multiplatform и Kotlin Multiplatform.
При планировании используй самые последние опубликованные версии библиотек и фреймворков.

Мы хотим построить работу поэтапно, начиная с разработки прототипа, который будет работать на MacOS, а затем добавим поддержку Windows и Linux.
После этого планируем добавить функциональность для настройки дефолтного браузера и поддержку различных типов ссылок.

Как должно работать
При запуске приложения оно добавляется в трей, но не открывается окно изначально.
При клике на иконку в трее, появляется меню возле иконки трея с пунктами "Настройки", "Завершить работу" (могут добавиться еще).
В настройках ОС при выборе дефолтного браузера, приложение должно отображаться в списке доступных браузеров.
Приложение должно уметь список всех приложений, которые могут открыть ссылки (веб, почта, может быть что-то еще с custom url scheme). 
Если установлено несколько версий одного и того же браузера, отображать их как разные браузеры с указанием версии.
При клике на какую-либо ссылку, должен открываться попап со списком выбора браузера. 
Если браузеров (включая разные версии одного и того же) больше трех - должно отображаться только три браузера, остальные скрыты в выпадающем меню по кнопке "Показать все"
По клику на браузер, ссылка должна быть открыта в нем.

Приложение должно поддерживать несколько тем (темная, светлая, системная), и возможность переключать язык интерфейса.

По клику в пункте меню трея на "Настройки", должно открыться окно настроек.
Экран настроек должен содержать следующие настройки: 
- Старт приложения при запуске ОС
- Список всех браузеров, установленных в системе, с возможностью сортировки и фильтрации (исключения этого браузера из списка доступных для открытия). Также там должна быть возможность добавления браузеров, которые могут открывать ссылки, но которые по каким-то причинам не были обнаружены самой системой.
- Настройка кастомных правил для открытия ссылок в определенных браузерах.
- Выбор темы, выбор языка.

Для платформенных особенностей и ограничений, сформируй список действий, которые необходимо сделать вручную, связанных с особенностями ОС.
Например, такой особенностью для MacOS будут сертификаты Apple, чтобы приложение было зарегистрировано как приложение, которое умеет открывать ссылки.
Например, для Windows может потребоваться регистрация приложения в реестре для работы с URL-схемами.

Если нужно использовать иконки, используй иконку заглушку, и в коде напиши TODO на замену и отдельно упоминай в разговоре каждый раз, что ты это сделал.

Мы собираемся реализовать этот проект с помощью нескольких ИИ агентов.
Предложи план, по которому мы сможем создавать этот проект поэтапно и параллельно.
Предложи архитектуру, которая будет учитывать указанные требования.

Начни с шага, где ты сделаешь минимально возможный работающий прототип, откуда мы сможем параллелить работу над конкретными фичами.
Этот прототип точно должен включать в себя архитектуру проекта, и само приложение на этом этапе должно уметь висеть в трее и показывать меню по клику на иконке в трей с двумя пунктами-заглушками - выход и настройки.
При работе над фичами, создавай новую ветку для этой фичи, покрывай весь функционал тестами и при каждом изменении запускай все тесты, чтобы убедиться в отсутствии регрессий. После этого собирай артефакт, и если все успешно, то коммити изменения в репозиторий на текущую ветку. (НЕ ПУШИТЬ)

Для каждого этапа, в каталог ai_stages (нужно создать 1 раз) создавай отдельную папку для этого этапа, и в ней создавай plan.md, в котором ты опишешь подробно, как этот этап должен быть реализован.
